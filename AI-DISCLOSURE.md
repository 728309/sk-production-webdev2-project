# AI Disclosure

AI was used as a development assistant while building SK Production Hub.

## How AI Was Used

AI support was used for planning, debugging, documentation, and final quality checks. Specific examples include:

- Planning the overall project structure for a Vue frontend and PHP REST API backend.
- Designing a readable Vue component structure with atoms, molecules, organisms, templates, and pages.
- Adding and reviewing Vue Router route structure and protected route behavior.
- Adding and reviewing Pinia authentication state for login, logout, current user, and admin checks.
- Planning the PHP Controller -> Service -> Repository backend organization.
- Debugging JWT authentication and role authorization issues.
- Debugging Git, Docker, npm, and PowerShell setup issues during development.
- Improving README files, setup instructions, API documentation, and test account documentation.
- Checking the project against the Web Development 2 rubric and assignment requirements.
- Comparing the project structure and documentation depth with Visual Shield as a high-grade benchmark.

## Student Responsibility

The student made the design and feature decisions for SK HUB, reviewed the generated suggestions, edited the code, tested the workflows, and kept responsibility for the final result. AI was not used to blindly submit code without understanding it.

The student should still be able to explain:

- How the Vue pages and reusable components are organized.
- How Vue Router protects authenticated and admin-only routes.
- How Pinia stores the token and current user.
- How frontend API modules call the PHP backend.
- How the backend controllers, services, and repositories work together.
- How password hashing, JWT validation, and role checks protect the app.
- How Docker starts nginx, PHP, MariaDB/MySQL, and phpMyAdmin.

## Important Learning Points

This project helped reinforce:

- Vue 3 components and page structure.
- Vue Router navigation and route guards.
- Pinia authentication state.
- Frontend API calls with JWT bearer tokens.
- PHP REST API routes and JSON responses.
- Controller -> Service -> Repository separation.
- Password hashing and JWT validation.
- Role-based authorization for admin endpoints.
- PDO prepared statements.
- Docker workflow with nginx, PHP, MariaDB/MySQL, and phpMyAdmin.
- Final submission polish, including documentation, setup guides, and demo flow.

## Visual Shield Benchmark Note

Visual Shield was inspected only as a quality benchmark for structure, documentation depth, naming clarity, accessibility explanation, and final submission polish. No Visual Shield code, text, domain logic, styling, component names, or files were copied into SK HUB.
